# Управление вакансиями

## Назначение

Эта страница используется для управления публичными настройками предопределенных вакансий на сайте.

Админ-страница: `/admin/vacancies`

Важно: вакансии не создаются динамически через админку. Система поддерживает только 3 предопределенные вакансии:

- `iosdev` -> iOS Developer
- `qaengineer` -> QA Engineer
- `designer` -> UI/UX Designer

## Что можно менять

Для каждой вакансии в админке можно:

- Показать или скрыть вакансию на публичной странице careers
- Включить или выключить метку **Hot position**
- Изменить отображаемый **Candidate level**

## Что нельзя менять из админки

На этой странице нельзя создавать новые вакансии и нельзя переименовывать существующие.

Следующие значения зафиксированы в коде и через эту страницу не редактируются:

- Vacancy ID
- Vacancy slug
- Vacancy title
- Conditions / tags
- Employment type
- Location

## Как обновить вакансию

1. Откройте `/admin/vacancies`.
2. Найдите нужную карточку вакансии.
3. В блоке **Change values** измените нужные настройки:
4. Переключите **Show on career page** в нужное состояние.
5. Переключите **Hot position** в нужное состояние.
6. Выберите значение в **Candidate level** или оставьте **Not specified**.
7. Нажмите **Save** в карточке этой вакансии.

После сохранения обновленные значения применяются к публичной карточке вакансии и к странице деталей вакансии.

## Доступные уровни Candidate level

- Intern
- Trainee
- Junior
- Middle
- Senior
- Lead
- Principal
- Head

Поле можно оставить пустым, выбрав **Not specified**.

## Логика видимости

- Если **Show on career page** включен, вакансия публично показывается.
- Если **Show on career page** выключен, вакансия скрыта из публичного career-раздела.

## Текущие URL вакансий

- `/career/iosdev`
- `/career/qaengineer`
- `/career/designer`

## Технический flow

### Как связаны админка, Firebase и публичный сайт

1. Администратор открывает `/admin/vacancies`.
2. Админка загружает настройки вакансий через `GET /api/admin/vacancies`.
3. При изменении вакансии админка отправляет защищенный `PATCH` запрос на `/api/admin/vacancies/{vacancyId}`.
4. Бэкенд проверяет admin-доступ и наличие permission `vacancies.manage`.
5. Бэкенд обновляет запись предопределенной вакансии в Firestore.
6. Публичные career-страницы читают подготовленные данные вакансий через backend и показывают только те вакансии, которые отмечены как published.

### Что хранится в Firebase

Для каждой управляемой вакансии Firebase / Firestore хранит редактируемые публичные настройки, например:

- `isPublished`
- `isHot`
- `level`
- служебные метаданные вроде `updatedAt` и `updatedBy`

Статические определения вакансий, такие как ID, slug, title, location, tags и employment type, определены в коде и объединяются с данными Firebase на бэкенде.

### Модель доступа

- Изменения из админки проходят через защищенные server API routes.
- Публичные пользователи не могут редактировать данные вакансий.
- Админка не создает произвольные vacancy-документы, а обновляет только поддерживаемые предопределенные vacancy ID.

## Примечания

- Эта страница управляет только публичными настройками 3 предопределенных вакансий.
- Если бизнесу нужна новая вакансия, ее сначала нужно добавить в код.
