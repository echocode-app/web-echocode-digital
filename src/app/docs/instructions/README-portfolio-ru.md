# Управление портфолио

## Назначение

Эта страница используется для добавления и удаления динамических preview-карточек в публичном разделе портфолио.

Админ-страница: `/admin/portfolio`

Важно: этот инструмент создает только preview-карточку портфолио. Полноценную страницу кейса он не создает.

## Как добавить карточку портфолио

1. Откройте `/admin/portfolio`.
2. В форме **Add preview project** заполните все 4 обязательных шага:
3. Укажите **Project title**.
4. Загрузите **Project image**.
5. Выберите минимум одну **Platform**.
6. Выберите минимум одну **Category**.
7. Нажмите **Create card**.

После сохранения новая карточка появится в списке в админке и в публичном превью портфолио.

## Обязательные поля

- Title
- Image
- Минимум одна platform
- Минимум одна category

Кнопка **Create card** остается неактивной, пока все обязательные поля не заполнены корректно.

## Доступные значения Platform

- iOS
- Android
- Web
- Unity

## Доступные значения Category

- E-commerce
- Utility
- iGaming
- Health & Well-being
- Education
- Lifestyle
- Creative
- Travel

## Требования к изображению

- Максимальный размер файла: `5 MB`
- Поддерживаемые форматы: `JPEG`, `PNG`, `WebP`, `GIF`, `AVIF`, `BMP`, `TIFF`, `HEIC`, `HEIF`
- Рекомендуемый размер изображения для лучшего визуального результата: `1024 x 1024` или другое изображение с похожей квадратной пропорцией

Если файл слишком большой или формат не поддерживается, админка покажет ошибку.

## Что делает система внутри

- Система автоматически генерирует безопасный внутренний ID на основе названия проекта.
- Сначала загружается изображение.
- После этого создается preview-карточка и связывается с загруженным изображением.

## Технический flow

### Как связаны админка, Firebase и публичный сайт

1. Администратор открывает `/admin/portfolio` и работает в UI админки.
2. Админка отправляет авторизованные запросы в admin API по маршрутам `/api/admin/portfolio`.
3. Бэкенд проверяет admin-доступ и наличие permission `portfolio.manage`.
4. Для новой карточки бэкенд сначала создает signed upload URL для Firebase Storage.
5. Изображение загружается в Firebase Storage.
6. После успешной загрузки бэкенд создает запись preview-карточки портфолио в Firebase / Firestore.
7. Публичный сайт читает опубликованные данные портфолио и показывает карточку в разделе portfolio.

### Что где хранится

- Файл изображения: Firebase Storage
- Метаданные preview-карточки: Firestore
- Публичный рендер карточки: клиентский сайт читает подготовленные данные через backend

### Модель доступа

- Admin write-действия доступны только через авторизованные admin API routes.
- Публичные посетители видят только тот контент, который отдается публичным website flow.
- Админка не пишет напрямую в Firebase из браузера; запись идет через защищенные серверные endpoints.

## Как удалить карточку портфолио

1. Откройте `/admin/portfolio`.
2. Найдите нужную карточку в списке.
3. Нажмите действие удаления.
4. Подтвердите удаление в диалоговом окне браузера.

Удаление убирает динамическую preview-карточку из админки и из публичного превью портфолио.
